# AHT20 + BMP280 高精度环境监测工程 (ESP32-S3)

本工程实现了 AHT20（温湿度）与 BMP280（气压/温度）传感器的深度集成，采用纯 Rust 编写的硬件抽象层（esp-hal 1.0）。

## 核心特性

- **本地原生驱动**：**特别说明**：虽然硬件接线保持不变，但由于现有的第三方在线库（Crates.io）在 `esp-hal 1.0` 环境下存在严重的协议时序兼容性问题（如 AHT20 的 CRC 校验死锁、BMP280 初始化握手失败），本项目通过实现本地手动驱动彻底解决了这些**应用层软件问题**。
- **高精度补偿算法**：完整实现了 BMP280 官方推荐的 64 位整数/浮点数混合补偿逻辑。
- **软件校准接口**：支持通过 `set_calibration_offsets` 函数实时修正传感器偏差（如抵消 CPU 发热影响）。
- **海拔换算**：内置简易气压-海拔转换函数。
- **总线共享**：使用 `embedded-hal-bus` 的 `RefCellDevice` 模式实现安全的一路 I2C 挂载多个设备。

## 硬件接线

两个传感器共享同一条 I2C 总线，且**必须外接上拉电阻**：

| 传感器信号 | ESP32-S3  | 备注                     |
| :--------: | :-------: | :----------------------- |
|  **VCC**   | **3.3V**  | 建议接 3.3V              |
|  **GND**   |  **GND**  | 共地                     |
|  **SDA**   | **GPIO4** | **须外接 4.7kΩ 到 3.3V** |
|  **SCL**   | **GPIO3** | **须外接 4.7kΩ 到 3.3V** |

## 快速运行

在工作区根目录执行：

```powershell
cargo run -p aht20-bmp280-app --release -- --port COMx
```

## 故障分析

如遇到测量数据不更新或读数为零，请重点检查：

1. **上拉电阻**：ESP32-S3 内部上拉电阻较弱，挂载多个设备时信号质量易下降。
2. **I2C 映射**：由于本项目避开了受干扰较多的 GPIO1/0，改用 GPIO4/3，请确保连线正确。
